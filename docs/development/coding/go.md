# Goコーディングガイドライン

## 構造

- パッケージは所有する責務で分け、循環依存を設計で避ける。
- `internal`で外部公開範囲を制限し、公開識別子を必要最小限にする。
- interfaceは利用側で定義し、テストのためだけに大きなinterfaceを作らない。
- `context.Context`は最初の引数で明示的に渡し、構造体へ保存しない。

## エラーと並行処理

- エラーを`%w`で包み、判断が必要なものは`errors.Is`または`errors.As`で識別できるようにする。
- goroutineの所有者、停止条件、チャネルを閉じる側を明確にする。
- タイムアウト、キャンセル、部分失敗をテストする。
- mutex保護範囲を小さくし、ロック保持中に外部I/Oを行わない。

## 検査

```sh
gofmt -w core
cd core && go test ./...
cd core && go vet ./...
```
